Sección 2 — Investigación: Calidad de Software
Fundamentos de calidad basados en fuentes bibliográficas reconocidas
1

Definición de Calidad de Software

La calidad del software es el grado en que un sistema, componente o proceso cumple los requisitos especificados y las necesidades o expectativas del cliente o usuario. Esta definición proviene del estándar IEEE 610.12 y es ampliamente adoptada en la industria.

Según la norma ISO/IEC 25010 (SQuaRE), la calidad del software se evalúa a través de características como: funcionalidad, fiabilidad, usabilidad, eficiencia, mantenibilidad y portabilidad. Un software de calidad no solo ejecuta correctamente sus funciones, sino que lo hace de manera eficiente, segura y satisfactoria para el usuario.

Roger Pressman en su libro Ingeniería del Software añade que la calidad es una gestión continua que involucra procesos, personas y productos.

Fuentes:

📚 IEEE Std 610.12-1990 · IEEE Standard Glossary of Software Engineering Terminology

📚 ISO/IEC 25010:2011 · Systems and software quality requirements and evaluation (SQuaRE)

📚 Pressman, R. (2010). Ingeniería del Software: Un Enfoque Práctico (7.ª ed.). McGraw-Hill.

2

Concepto de Usabilidad en Software

La usabilidad es la capacidad de un producto de software para ser entendido, aprendido, usado y resultarle atractivo al usuario cuando se utiliza en determinadas condiciones. Es una de las subcaracterísticas de calidad definidas en ISO/IEC 25010.

Jakob Nielsen, pionero en usabilidad, la descompone en cinco componentes: 1) Facilidad de aprendizaje (¿qué tan fácil es para los usuarios realizar tareas básicas la primera vez?), 2) Eficiencia (una vez aprendido, ¿con qué rapidez realizan las tareas?), 3) Memorización, 4) Errores (cuántos cometen y qué tan graves son), y 5) Satisfacción (¿les agrada el diseño?).

En el contexto de las pantallas desarrolladas en la Sección 1, la usabilidad se aplica mediante: etiquetas claras, mensajes de error descriptivos, flujo lógico de campos, y una paleta de colores coherente que facilita la navegación.

Fuentes:

📚 Nielsen, J. (1993). Usability Engineering. Academic Press.

📚 ISO/IEC 25010:2011 · Característica de Usabilidad

📚 Hassan Montero, Y. y Martín Fernández, F.J. (2003). Qué es la Usabilidad. No Solo Usabilidad Journal.

3

Estándares de Calidad de Software

Los estándares internacionales de calidad de software establecen marcos de referencia para evaluar, certificar y mejorar los procesos de desarrollo. Los principales son:

ISO/IEC 25010 (SQuaRE): Define el modelo de calidad del producto de software con 8 características principales: adecuación funcional, eficiencia de desempeño, compatibilidad, usabilidad, fiabilidad, seguridad, mantenibilidad y portabilidad.

ISO/IEC 12207: Establece los procesos del ciclo de vida del software, incluyendo desarrollo, mantenimiento y operación con criterios de calidad en cada etapa.

CMMI (Capability Maturity Model Integration): Modelo de madurez que evalúa los procesos de desarrollo de software en 5 niveles, desde inicial (1) hasta optimizado (5), guiando a las organizaciones hacia la mejora continua.

Fuentes:

📚 ISO/IEC 25010:2011 · SQuaRE – System and Software Quality Models

📚 ISO/IEC 12207:2017 · Software and systems engineering — Software life cycle processes

📚 CMMI Institute (2018). CMMI for Development v2.0. Software Engineering Institute – Carnegie Mellon.

Tecnologías para Implementar Calidad en Software

Existen diversas herramientas y tecnologías que permiten implementar y garantizar la calidad durante el ciclo de desarrollo del software:

SonarQubeAnálisis estático de código y detección de errores
SeleniumPruebas automatizadas de interfaces de usuario (UI)
JUnit / TestNGFrameworks de pruebas unitarias en Java
JenkinsIntegración continua (CI/CD) y automatización
JIRAGestión de bugs, tareas e incidencias
PostmanPruebas de APIs REST y validación de servicios
FigmaPrototipado y validación de usabilidad (UX)
DockerEntornos reproducibles y pruebas aisladas

Estas herramientas se integran en metodologías ágiles como Scrum y XP (Extreme Programming), que incluyen prácticas de calidad como revisiones de código, pruebas de regresión y refactorización continua.
